## Headline: DARK Tunnel is not proprietary — it's now decoded

Every prior session recorded `.dark` as "proprietary encryption, no
public decryptor" (and the orphaned `dark_parser.py` repeated it). **That
was never checked against the bytes.** A `.dark` file is:

```
darktunnel://<base64(JSON)>
```

where the JSON is plaintext. It always exposes `type`
(VLESS/VMESS/TROJAN/…), `name`, and transport hints; on *unlocked*
configs it carries the server fields directly. What's actually encrypted
is the optional `encryptedLockedConfig` field — DARK Tunnel's "locked
config" DRM, sealed with a key not present in the file.

Shipped **scheme I1** (`decrypt/dark_decrypt.py`, `ac2eed1`): tolerant
base64 (standard + URL-safe) → JSON, extract everything exposed, report
`PARTIAL` + a clear "locked by author" warning when the credential body
is sealed, `SUCCESS` otherwise. Wired through the router (+ a fix so the
router keeps `PARTIAL` payloads instead of dropping them as `FAILED`), a
`_normalize_dark` normalizer, `/api/formats` (DARK now `decryptable: I1`),
and 5 tests over the bundled samples. Removed the dead `dark_parser.py`.
All 4 DARK samples now decode to protocol + name + transport (suite
43→48).
