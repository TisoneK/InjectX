## Root cause of "messed up decoded page"

Ran the app end-to-end (started the backend, served the frontend, injected
real decoded configs into `renderConfigDetail`). Two findings:

1. **The screenshot's EHI showed EXTRACTION FAILED, but a fresh backend
   decodes it fine** (EHI 6/6, HC 13/13, ZIV 6/6, DARK 4/4 partial). The
   user's running app was on a **stale backend process** predating this
   week's fixes — a restart fixes it.
2. **Even on success, the detail view was near-empty.** The structured
   sections only render fields mapped to IR slots; formats like HTTP
   Injector and ZIVPN keep ~all their data in `raw_data._all_fields`. A
   decoded EHI rendered exactly ONE row (SNI) despite 21 extracted
   fields — hence "export JSON to read it." (Backlog N9.)

**Output redesign:**

  raw-JSON toggle never collapsed. Added the utility.
