## Result: ZIVPN cracked — `.ziv` decodes 6/6 (was 0/6)

The "key rotated, not publicly reversed, unfixable" verdict that stood
across Sessions 9–14 was **wrong**. The key was extractable from the APK
in under an hour of static analysis. `f6e91d8` ships it.

### What the app actually does (ZIVPN Tunnel v2.1.5, com.zi.zivpn)
Static analysis (androguard, no JVM) of the config importer:
- `ConfigImportFileActivity.onActivityResult` → background thread → helper
  `Lo3/a;->b(Context, InputStream)`.
- `Lo3/a;.<clinit>` builds the password by base64-decoding five constants
  `U2VjdXJlUGFydDE=`..`U2VjdXJlUGFydDU=` → `"SecurePart1".."SecurePart5"`,
  concatenated: **`SecurePart1SecurePart2SecurePart3SecurePart4SecurePart5`**.
- `Lu3/c;.a(String)` splits the payload on `.` → `salt.iv.ct`, base64-
  decodes each, derives an AES key via `Lv3/b;` (BouncyCastle PKCS5S2 =
  PBKDF2, **1000 iterations**, 16-byte key), and AES-GCM decrypts with
  `IvParameterSpec(iv)`.
- Output: ZIVPN XML `<entry key="…">value</entry>` properties.

### Why it fooled prior sessions
InjectX's H1 algorithm — `PBKDF2(SHA256, dkLen=16, count=1000)` + AES-GCM
over `salt.iv.ct` — was **already exactly correct**. Only the password
was stale (`fubvx788b46v` → the SecurePart string). Adding one password
made all 6 samples decode. A red herring along the way: the app has no
literal `pbkdf2`/`PBEKeySpec` string (BouncyCastle's PKCS5S2 uses its own
obfuscated classes), which briefly suggested "no PBKDF2 in the app" — the
lesson is to follow the config-import call graph, not string-grep for
algorithm names.

### Also fixed

## Methodology lesson (logged as an inefficiency)
"Blocked on external key material" ≠ "unfixable." It means *nobody has
extracted it yet*. For an app-locked config format, the key lives in the
APK and static analysis of the import path recovers it. Five sessions
deferred this as impossible; it took one APK + androguard.

## How the APK was obtained
Automated mirror download failed (Session 15 first half: APKPure 403,
Uptodown served its own installer, APKCombo JS-gated). The **user
supplied** the XAPK. Extraction ran locally: `unzip` the XAPK → base
`com.zi.zivpn.apk` → androguard (pip, no JVM — the machine's `/usr/bin/java`
is a stub; jadx couldn't run). This is the repeatable path.
