# InjectX Review — 2026-07-15 (Session 16)

**Agent:** Claude Code (claude-fable-5), local macOS · **Role:** engineer
**Task:** Extract the current TLS Tunnel (`.tls`) AES-GCM key from the app
APK, same method that cracked ZIVPN. User supplied TLS Tunnel v8.0.6 APK
(`com.tlsvpn.tlstunnel`).

## Result: blocked by DexProtector — no code change

Unlike ZIVPN (unprotected, cracked in minutes), **TLS Tunnel 8.0.6 ships
DexProtector** — a commercial anti-tamper/packer that encrypts the app's
own strings and classes and only decrypts them at runtime. Static key
extraction is defeated by design.

### What I did
- Extracted the APK; confirmed `com.tlsvpn.tlstunnel`.
- androguard static analysis: enumerated **every** class referencing
  `AES/GCM` / `SecretKeySpec` in all 4 dex. All hits were unprotected
  third-party SDKs, none the config crypto:
  - `Lhm;` → Digital Turbine "Ignite" ad SDK (`OneDTAuthenticator`,
    `dtx_ignite_service_storage`, AndroidKeyStore).
  - `Lo30;` → Conscrypt TLS provider.
  - `zzhkh` (Google Tink), `mbridge` (Mintegral ad SDK).
- No `com.tlsvpn.tlstunnel` class references `SecretKeySpec`/`GCM`
  directly — the app's crypto is DexProtector-hidden.
- Native libs present: **`libdexprotector.so`** (the packer) and
  `libtlsvpn.so` (app native code).
- Brute-forced **148,232** unique strings (all dex + all `.so`) × 5 key
  derivations (raw bytes, SHA-256, MD5, SHA-256[:16], base64-decode) as
  AES-128/192/256-GCM against a real sample. **No hit.**

### Why static fails here
DexProtector encrypts the app's string pool and class code; only ad
SDKs (which the packer leaves alone) expose plaintext crypto. The real
config key is materialized at runtime, so it is not in any static
string, and it is not a simple derivation of one.

## The path forward for TLS: dynamic Frida on a device
This is exactly the case Method 2 in `docs/key-extraction.md` exists for.
Hook `javax.crypto.SecretKeySpec.<init>` / `Cipher.doFinal` (or the
native `libtlsvpn.so` via `libcrypto` hooks) on a rooted device or
emulator, import a `.tls` config in the app, and read the key off the
Frida console *after* DexProtector has decrypted it. This machine has no
emulator/rooted device, so it needs the user's device. I can provide the
Frida script.
