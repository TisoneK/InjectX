## What shipped

**`feat(decrypt)`: runtime keyfile via `INJECTX_KEYFILE`** (`29628d9`).
`KeyStore._load_keyfile()` and `get_router(keyfile_path)` already
existed, but `parse_config` called `get_router()` with no path — so the
external-keyfile mechanism was **dead code**: no runtime way to supply a
key, every key hardcoded. `get_router()` now falls back to the
`INJECTX_KEYFILE` env var (matching the `INJECTX_HOST/PORT/UPLOAD_DIR`
convention). A JSON file keyed by KeyStore category merges over the
built-in defaults. This is the drop-in path for keys we extract
ourselves. +2 regression tests (merge + env wiring); suite 41→43.

**`docs/key-extraction.md`.** The paired half: how to recover current
keys ourselves (static jadx/apktool grep for `SecretKeySpec`/`PBKDF2`;
dynamic Frida hook on `SecretKeySpec.$init`/`Cipher.doFinal`), and how
to load them via the keyfile. Explicit that there is no keyless
shortcut — AES-GCM without the key is unrecoverable.

## Why this is the right "possible now" increment

The blocker on TLS/ZIV is external key material, not algorithms (Session
12). Nothing in-repo decodes them until a current key surfaces. The
highest-leverage thing achievable **without the target APKs** is to make
InjectX able to *accept* such a key the instant it's found — turning a
source edit + rebuild into an env var. Running the extraction itself
needs the APKs (and an emulator/Frida for the dynamic path), which this
environment can't do.

## Side question logged (not implemented): WPA `.cap` 4-way handshake
Different domain from the config decryptors (802.11/pcap, not app config
files). You don't "decrypt" the handshake — you use its nonces + MIC to
**crack the WPA/WPA2-PSK passphrase** offline (`hcxpcapngtool` →
`hashcat -m 22000` / aircrack-ng), then optionally decrypt the capture's
data frames once you have the passphrase. Same keyless-is-hopeless
reality: recoverable only if the passphrase is weak/in a wordlist. Would
be a **separate module** (pcap ingestion + a cracking backend), not an
extension of the existing format decryptors. Not started — it's a scope
decision for the user, captured here so it isn't lost.
