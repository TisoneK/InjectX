# InjectX Review — 2026-07-16 (Session 21)

**Agent:** Claude Code (claude-fable-5), local macOS · **Role:** engineer
**Task (user, from a terminal screenshot):** (1) Terminal has no copy
button; (2) the system uses the bundled assets as the source of truth —
asset configs are dev-only; (3) the terminal is "centered to assets"
instead of the user's file selection — `targets info "<windows path>"`
failed.

## Root causes

- The failing command `targets info "C:\Users\tison\Downloads\Telegram
  Desktop\Airtelke (3).hc"` broke because (a) the tokenizer split on
  whitespace, so the quoted path became just `"C:\Users\tison\Downloads\
  Telegram`, and (b) `info`/`debug` only resolved against loaded targets.
- The 31 "assets" were NOT auto-loaded (main.js sets no INJECTX_AUTOIMPORT
  — that env var is only used by dev/CI backend launches). They came from
  the prominent sidebar **Import Assets** button, which loads the bundled
  dev fixtures.

## What shipped (`feat(ui)` `f1f6dbe`)

- **Terminal copy:** COPY (whole output) + CLEAR buttons in the terminal
  toolbar, and a hover copy button per line (matches the activity log /
  Logs page). `termWrite` lines are now flex rows with a `.term-copy`.
- **Quoted-arg tokenizer** (`tokenize`): honours "double"/'single' quotes
  so paths with spaces stay one token.
- **Path-aware inspection** (`resolveOrParse` + `looksLikePath`):
  `targets info`/`targets debug` accept a loaded id/filename OR a
  filesystem path — a path that isn't already loaded is parsed fresh via
  `API.parseConfig` (backend reads the real file) and added to the list.
- **`targets pick`:** opens the native OS file dialog
  (`API.openFileDialog`) and imports the selected config(s) — the
  terminal path to "select from my machine".
- **Assets dev-only:** exposed `app.isPackaged` as `is-dev` (IPC →
  preload `isDev` → `API.isDev`), and the renderer hides the sidebar
  "Import Assets" button when not dev (packaged app). Tooltip now says
  DEV ONLY. It still shows when running via `electron .`.

## Verification
