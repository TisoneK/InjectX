## What shipped (`feat(app)` `cb71d75`)

An IDE-style folder workflow with persistence, mostly Electron main:

- **Settings store** (`frontend/main.js`): a small `settings.json` under
  `app.getPath("userData")` holding `lastDir` and `lastFolder`, loaded in
  `whenReady`, written on change (`rememberDir`/`rememberFolder`).
- **Pickers remember location:** `open-file-dialog` and `handleOpenFile`
  now seed `defaultPath` from `lastDir` and record the chosen directory.
- **Folder support:** new `open-folder-dialog` (native directory picker),
  `list-config-files <folder>` (fs scan filtered by `CONFIG_EXTENSIONS`),
  and `get`/`set-last-folder`. Exposed via preload + api.
- **Renderer:** `importFolder(folder, quiet)` (list → parse each →
  reload), `openFolder()` (dialog → import). A sidebar **Open Folder**
  button. Terminal `targets openfolder` and `targets import <folder>`
  (a path whose last segment has no dot is treated as a folder).
- **Reopen last project:** on startup, after `loadConfigs`, if a last
  folder is remembered it is silently re-imported — the backend's
  in-memory store is cleared on restart, so this restores the user's
  working set automatically.
