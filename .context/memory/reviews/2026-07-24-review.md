## Research Findings (what the web-search pass turned up)

The user's initial research context was accurate as of when it was written but had two gaps the search pass closed:

1. **ECH has graduated from proposal to published standard.** The user's context called ECH a "proposal." It is now **RFC 9849** (published March 2026), with OpenSSL 4.0 and NGINX having landed ECH support. The SNI inspection window ISPs exploit is closing — but not closed yet. The design doc treats SNI inspection as a "technique with a sunset" and designs the discovery half (CT logs) independent of the verification half (active probing) so the feature degrades gracefully when ECH kills SNI inspection.

2. **Two reference implementations exist that the user's context didn't mention.** Both are directly transferable to InjectX:
   - **BugScanX** (`pddung93/BugScanX`, `pip install bugscan-x`, MIT) — CLI tool with 8 menu items (subdomain scanner, IP scanner, subdomain finder, reverse IP, host OSINT, TXT toolkit, open port checker, DNS records). Same dependency profile as InjectX already has.
   - **SNIbugtester** (`Praveenhalder/SNIbugtester`, MIT) — Android app whose UX is the best reference for InjectX's UI: multi-file loading, range selection, thread control (default 200), timeout control (default 5s), cloudflare-only filter, live progress bar, results table color-coded by HTTP status, IPs tab, .txt/.csv export. Essentially the spec InjectX should implement.

3. **The African ISP zero-rating context was grounded.** Safaricom (Free Basics, ~69% Kenya market share), Airtel Kenya (also Free Basics), Telkom (zero-rated educational portals) are all confirmed. Public bug-host lists circulate at Techfoe, aimtuto.com, snihost.com, and Scribd — these are the seed-list sources the implementing session should curate into `data/seedlists/{safaricom-ke,airtel-ke,telkom-ke}.txt`.

Full reference list (30+ primary sources) is in §11 of the design doc.
