## Diagnosis of "only HC works as desired"

The user runs on **Windows** (`C:\Users\tison\...` in the export paths).
Their four exports:

| Format | Export status | What it contained |
|--------|---------------|-------------------|
| HC | success | host, port, ssh_server/port/user/pass, proxy_host, payload — full |
| EHI | **failed** | nothing |
| ZIV | success | host, protocol, remote_dns — only 3 fields |
| DARK | partial | protocol only |

Two separate causes:

1. **Sparse exports** — the `/api/config/export` endpoint dropped
   `raw_data` as "too verbose". But HC is the *only* format that maps its
   data to top-level IR fields; HTTP Injector, ZIVPN, and DARK keep it in
   `raw_data._all_fields`. So dropping raw_data made every non-HC export
   look empty. **This is the "only HC works" cause for exports.**
2. **EHI failed on the user's machine** — scheme B2 needs `argon2-cffi`.
   On this Mac's synced venv EHI decodes 6/6; on the user's Windows
   backend venv the dep is almost certainly missing/stale, so B2 throws
   and falls back to FAILED. Environment, not code — the user needs to
   reinstall backend deps (`pip install -r backend/requirements.txt`).
